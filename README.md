# News Deduplication Pipeline

Проект для поиска и кластеризации дубликатов новостных статей. Алгоритм спроектирован для работы в **полностью автономном (offline) режиме** и использует современные нейросетевые подходы (LLM Embeddings & Cross-Encoders).

## Архитектура решения

Поиск дубликатов реализован по золотому стандарту индустрии (Two-Stage Pipeline), что решает проблему ложных срабатываний ("слипания" похожих по тематике, но разных по фактам новостей):

1. **Этап 1: Векторный поиск (Bi-Encoder + FAISS)**
   - **Модель:** `multilingual-e5-base` (или аналогичная).
   - **Суть:** Каждая новость превращается в плотный числовой вектор (эмбеддинг). FAISS очень быстро (за миллисекунды) находит все потенциально похожие пары по косинусному сходству с пониженным порогом (например, `0.90`).
   - **Цель:** Обеспечить 100% Recall (не потерять ни одного истинного дубликата, даже если абзацы перемешаны).

2. **Этап 2: Реранжирование (Cross-Encoder)**
   - **Модель:** `BAAI/bge-reranker-v2-m3` (или аналогичная).
   - **Суть:** Все подозрительные пары, найденные на первом этапе, передаются в Cross-Encoder. Модель читает оба текста одновременно и выдает строгую оценку от 0 до 1, насколько один текст логически вытекает из другого (Entailment / Similarity).
   - **Цель:** Исключить ложные срабатывания (False Positives), отбросив новости о разных событиях с похожей лексикой (например, финансовые отчеты разных компаний).

3. **Этап 3: Графовая кластеризация (NetworkX)**
   - Подтвержденные пары объединяются в граф (с учетом временного окна в 3 дня).
   - Компоненты связности (Connected Components) образуют финальные кластеры событий.

## Структура проекта

* `data/` — Директория с данными:
  * `news_dump_full.json` — Исходный "сырой" дамп новостей.
  * `news_dump_unique.json` — Очищенный от 100% текстовых дубликатов дамп (генерируется `extract.py`).
  * `synthetic_duplicates_clean.json` — Синтетический датасет для проверки метрик качества (создается `generate_nlp_duplicates.py`).
  * `clusters_output.json` — Итоговый результат кластеризации.
* `models/` — Директория для хранения офлайн-моделей (скачиваются отдельно или клонируются через `git clone`).
* `extract.py` — Скрипт для предварительной жесткой дедупликации (оставляет только уникальные тексты).
* `generate_nlp_duplicates.py` — Скрипт генерации синтетических дубликатов (перемешивание предложений, удаление слов) для валидации работы пайплайна через метрику Adjusted Rand Index (ARI).
* `news_dedup_pipeline.py` — **Основной скрипт** работы пайплайна.

## Установка и требования

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Подготовьте модели (Офлайн-режим):
   Склонируйте необходимые модели с HuggingFace в папку `models/`:
   ```bash
   git lfs install
   git clone https://huggingface.co/intfloat/multilingual-e5-base models/multilingual-e5-base
   git clone https://huggingface.co/BAAI/bge-reranker-v2-m3 models/bge-reranker-v2-m3
   ```

## Запуск

Для запуска основного пайплайна выполните:

```bash
python news_dedup_pipeline.py --model_path models/multilingual-e5-base --reranker_path models/bge-reranker-v2-m3 --limit 20000 --threshold 0.90 --cross_threshold 0.5
```

### Параметры:
- `--model_path`: Путь к папке с моделью эмбеддингов (Bi-encoder).
- `--reranker_path`: Путь к папке с моделью Cross-Encoder.
- `--limit`: Ограничение количества новостей для обработки (удобно для тестов).
- `--threshold`: Порог косинусного сходства для FAISS (Этап 1). Рекомендуется `0.90` при использовании реранкера.
- `--cross_threshold`: Порог уверенности для Cross-Encoder (Этап 2).

## Оценка качества (ARI)
При запуске пайплайн автоматически ищет поле `true_cluster_id` (присутствует в синтетических данных) и рассчитывает метрику **Adjusted Rand Index**. Это позволяет убедиться, что алгоритм не "ломается" при обновлениях моделей или порогов.
