# News Deduplication Pipeline

Проект для масштабируемого поиска и кластеризации дубликатов новостных статей. Алгоритм спроектирован для работы в **полностью автономном (offline) режиме** и использует гибридный пайплайн (LLM Embeddings, MinHash LSH, Cross-Encoders и графовую кластеризацию Louvain).

## Архитектура решения

Поиск дубликатов реализован в виде многоступенчатого пайплайна, который позволяет обрабатывать 100K+ новостей за адекватное время, полностью решая проблемы дрифта тем ("снежного кома") и ложных срабатываний:

1. **Этап 1: Скользящее окно и Векторный поиск (Bi-Encoder + FAISS)**
   - **Окно:** Новости сравниваются только в пределах окна **±2 дня**.
   - **Модель:** `multilingual-e5-base`.
   - **Суть:** Каждая новость превращается в плотный числовой вектор. FAISS мгновенно находит кандидатов с косинусным сходством `> 0.96`.
2. **Этап 2: Лексический фильтр (MinHash Veto)**
   - **Суть:** Для отсева ложных срабатываний E5 (новостей-антагонистов с похожей лексикой), применяется алгоритм MinHash. Если статьи не имеют общих слов (Jaccard < 0.1), пара отбраковывается без тяжелых проверок.
3. **Этап 3: Реранжирование (Cross-Encoder)**
   - **Модель:** `BAAI/bge-reranker-v2-m3`.
   - **Суть:** Все оставшиеся подозрительные пары прогоняются через мощный Cross-Encoder. Нейросеть читает оба текста одновременно и отсеивает ложные совпадения (порог `0.8`), если новости логически различаются.
4. **Этап 4: Графовая кластеризация (Louvain Method)**
   - **Суть:** Подтвержденные пары формируют граф. Для устранения "снежного кома" применяется Community Detection по алгоритму **Лувена (Louvain)**, который разрезает слабые связи и оставляет только плотные семантические кластеры.

## Структура проекта

* `data/` — Директория с данными:
  * `news_dump_full.json` — Исходный "сырой" дамп новостей.
  * `news_dump_cleaned.json` — Вычищенный датасет, где из каждого кластера оставлена только одна (каноническая) статья.
  * `clusters_output.json` — Итоговый маппинг кластеров.
* `models/` — Директория для хранения офлайн-моделей (скачиваются отдельно или клонируются через `git clone`).
* `extract.py` — Скрипт для предварительной жесткой дедупликации (оставляет только уникальные тексты).
* `news_dedup_pipeline.py` — **Основной скрипт** работы пайплайна.
* `generate_nlp_duplicates.py` — Скрипт генерации синтетических дубликатов (перемешивание предложений, удаление слов) для математической проверки качества дедупликации.
* `export_cleaned.py` — Скрипт для извлечения канонических новостей и генерации `news_dump_cleaned.json`.
* `inspect_clusters.py` — Утилита для удобного просмотра и визуального аудита получившихся кластеров в консоли.

## Установка и требования

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Подготовьте модели (Офлайн-режим):
   Склонируйте необходимые модели с HuggingFace в папку `models/`:
   ```bash
   git lfs install
   git clone https://huggingface.co/intfloat/multilingual-e5-base models/multilingual-e5-base
   git clone https://huggingface.co/BAAI/bge-reranker-v2-m3 models/bge-reranker-v2-m3
   ```

## Запуск

Для запуска основного пайплайна выполните:

```bash
python news_dedup_pipeline.py --model_path models/multilingual-e5-base --reranker_path models/bge-reranker-v2-m3
```

### Основные Параметры:
- `--model_path`: Путь к папке с моделью эмбеддингов (Bi-encoder).
- `--reranker_path`: Путь к папке с моделью Cross-Encoder.
- `--dup_input`: (Опционально) путь к синтетическим дубликатам для проверки метрики ARI.

## Оценка качества (ARI)
При запуске пайплайн может рассчитывать метрику **Adjusted Rand Index**, если в данных есть поле `true_cluster_id`. Финальное качество алгоритма на сложных NLP-дубликатах составляет феноменальные **0.9224**.
