# News Deduplication Pipeline (Senior-Level Architecture)

Масштабируемый пайплайн для дедупликации и кластеризации новостей с использованием FAISS и графовых алгоритмов (100% Offline).

1. **[README.md](README.md)** — Текущий файл. Быстрый старт, описание структуры проекта и запуск.

## Архитектура решения

1. **Embedding Generation**: Тексты новостей переводятся в плотные векторы (embeddings) с помощью модели `cointegrated/rubert-tiny2`, оптимизированной для русского языка.
2. **FAISS & Graph Clustering**: Быстрый векторный поиск соседей с асимптотикой `O(N log N)`. Поиск связанных компонент (Connected Components) через `networkx` объединяет похожие новости в кластеры событий.
3. **Time Window Heuristic**: В алгоритме реализовано временное окно (3 дня). Ребра в графе не создаются между новостями, опубликованными с разницей более 3 дней, что предотвращает склейку похожих, но разнесенных во времени событий (например, "ЦБ повысил ставку" в разные годы).

## Запуск

1. **Клонирование репозитория проекта**:
   ```bash
   git clone https://github.com/cavil2015/news_ded.git
   cd news_ded
   ```

2. **Загрузка модели (Offline)**:
   Чтобы скрипт работал на 100% локально и не скачивал ничего из интернета во время исполнения, необходимо скачать веса модели с Hugging Face в папку `models/`. Убедитесь, что у вас установлен [Git LFS](https://git-lfs.com/).
   ```bash
   mkdir models
   cd models
   git lfs install
   git clone https://huggingface.co/cointegrated/rubert-tiny2
   cd ..
   ```

3. **Зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Данные**:
   Положите данные (`news_dump_full.json` и `synthetic_duplicates.json`) в папку `data/`. Для работы метрики оценки качества (Adjusted Rand Index) используйте поле `true_cluster_id`.

4. **Запуск скрипта**:
   Для запуска на всем объеме данных:
   ```bash
   python news_dedup_pipeline.py
   ```
   
   Для быстрого тестирования пайплайна на ограниченной выборке (например, 100 новостей):
   ```bash
   python news_dedup_pipeline.py --limit 100
   ```

5. **Результат работы**:
   Скрипт формирует файл `data/clusters_output.json`, содержащий:
   - `cluster_id` — идентификатор группы
   - `canonical_news_id` — ID канонической (самой ранней) новости в кластере
   - `news_ids` — список всех новостей, относящихся к событию
